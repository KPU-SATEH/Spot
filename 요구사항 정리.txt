일반 요구 사항
 - 연료 감소를 위해 노력하는 운전자 비율 
 - 운전자의 대상자를 명확히 할 것 (분류 : 차종, 직종(트럭기사 택시기사 버스 기사))

거치대 관련
 - 카메라는 정중앙에 와야함
 - 시야에 방해되지 않는 장소에 있어야함
 - 핸드폰으로 네비게이션을 사용하는 인원을 위해 핸드폰을 보기 용이한 장소에 놔야함

일반 사용
 - 운행 시작시 어플 작동
 - 운행 종료시 운행 평가
 - 운행 종료시 서버로 해당 운행 데이터 전송한다
 - 운행 중에 백그라운드에서도 동작 가능 하다
 - 앱 내에서 운행, 일주일, 한달 통계(에코, 난폭, 졸음)를 볼수 있다

에코드라이빙
 - 급가속, 급정지, 급출발 횟수 통계

난폭 운전
 - 과속, 잦은 차선변경 을 기준으로 확인(기준 다시 잡을 필요 이씀)
 - 동작 중 위 기준을 여러번 충족시키면 경고 알람

졸음운전
 - 운전 시 앞차와의 거리가 지나치게 가까워지거나 충돌할 위험이 있을시 
 - 핸들을 일정시간 이상으로 움직이지 않을 시 (졸음운전의 경우 핸들을 고정시켜놓고 냅둠)
 - 핸들을 횡방향으로 과하게 움직일 시(졸음운전의 경우 좌우로 움직임)
 - 차선 변경이후 한번 더 차선이 움직일 시
 - 차선의 정중앙을 기준으로 많이 한쪽으로 치우쳐졌을 시
 - 평균 속도를 기준으로 위아래로 변화폭이 많아질 시 (엑셀을 누루는 힘이 달라짐)
 - 오래 운전할 시 운전자에게 휴식 경고 표시
 - 운전자의 난폭운전이나 의도적인 운전때문에 졸음운전을 캐치하기 힘듦 

서버
 - 계속 대기하며 기다리다가 운행 끝, 통계 요청 시 연결
 - 운행이 끝날때마다 해당 운행 데이터를 받음(에코, 난폭, 졸음 횟수?)
 - 사용자가 운행, 일주일, 한달 통계 요청시 데이터 전송

데이터 베이스(MYSQL)
 - 사용자번호, 날짜, 운행번호, 급가속, 급정지, 급출발, 에코드라이빙 점수, 난폭운전, 졸음 운전 포함
 - 사용자 번호와 날짜, 운행번호를 이용하여 각 운행을 구분
 - 사용자 번호 : 해당 사용자의 휴대폰 번호
 - 날짜 : 해당 운행의 날짜
 - 운행번호 : 해당 날짜에 운행한 번호(날짜마다 1부터 시작)
